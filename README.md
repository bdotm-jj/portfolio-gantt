# Portfolio Gantt ‚Äî Planned vs. Actual

A development-portfolio timeline for the pod operations meeting. Shows every active
project's **planned vs. actual** dates stacked on one row, ordered by the **Manual
Ordering** priority list, grouped by **line of business ‚Üí pod**, with an **On Deck**
queue and a **Backlog** parking lot beneath it.

The page itself is plain HTML/CSS/JS ‚Äî no React, no build step. It reads its data from
`data.js`, which a **scheduled GitHub Action refreshes straight from the Smartsheet API**.
So it's live (auto-updating) without exposing any token in the browser.

- `fetch-smartsheet.js` pulls both sheets, joins them on project name, and rewrites `data.js`.
- `.github/workflows/refresh-data.yml` runs that script on a schedule (hourly, Mon‚ÄìFri by
  default) and commits `data.js` when it changes. You can also run it on demand from the
  **Actions** tab.
- The browser only ever loads static files ‚Äî the Smartsheet token never leaves the Action.

The footer shows when the data was last refreshed.

## One-time setup (Lewis)

1. **Create a Smartsheet API token:** Smartsheet ‚Üí account menu ‚Üí *Personal Settings ‚Üí
   API Access ‚Üí Generate new access token*. Copy it.
2. **Add it as a repo secret:** GitHub repo ‚Üí *Settings ‚Üí Secrets and variables ‚Üí Actions
   ‚Üí New repository secret*. Name it exactly **`SMARTSHEET_TOKEN`**, paste the token.
3. **Enable Pages:** *Settings ‚Üí Pages ‚Üí Deploy from a branch ‚Üí `main` / root*.
4. **Kick off the first refresh:** *Actions ‚Üí Refresh Smartsheet data ‚Üí Run workflow.*
   That regenerates `data.js` from live data and commits it; Pages publishes within a minute.

That's it ‚Äî after that it refreshes itself on the schedule. To change cadence, edit the
`cron` line in the workflow.

> The token needs read access to both sheets:
> Portfolio Summary (`8767740555775876`) and Projects Intake (`4414567862980484`).

## How to read it

- **Hollow bar** = planned dates (write-once at intake). Its right edge is the go-live reference.
- **Solid bar beneath it** = actual dates, colored by line of business.
- **Red segment** = actual ran *past* the planned end (overdue).
- **Round badge** = Manual Ordering (`0` = in flight, `1, 2, 3‚Ä¶` = the queue).
- **Muted + hatched bar** with an "On Hold" / "Not Started" tag = not actively moving.
- **Red vertical line** = today. Calendar starts **1/1/2026**.
- **Yellow banner** lists any project whose planned dates are missing or invalid.
- **On Deck** = the forward queue per line of business.
- **Backlog** = submitted work not yet surfaced above.

## Local dry-run (optional)

To regenerate `data.js` on your own machine without waiting for the Action:

```bash
export SMARTSHEET_TOKEN=xxxxxxxx
node fetch-smartsheet.js
```

Node 18+ (it uses the built-in `fetch`).
